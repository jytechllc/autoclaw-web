# AutoClaw Growth Ops Cadence

## Objective

Create one operating rhythm for outbound, SEO, GEO, and social so work is visible, measured, and improved weekly.

## Weekly Rhythm

### Monday

- review last week's tracker
- pick one ICP and one offer angle for this week
- decide one GEO page or page update
- decide one X post, one LinkedIn post, one short-form video

### Tuesday

- enrich and send the next outbound batch
- publish or update one public GEO-friendly page
- refresh homepage or CTA copy only if replies show mismatch

### Wednesday

- send first follow-up batch
- post social proof, workflow content, or direct-offer post
- review booked calls and reply quality

### Thursday

- build sample lead packs for warm prospects
- review page clicks, conversions, and top-performing messages
- note what objections repeated this week

### Friday

- send second follow-up batch where appropriate
- write a short weekly growth memo
- decide what to stop, continue, and change next week

## Metrics Review

Track these every week:

- new priority companies added
- contacts enriched
- first-touch emails sent
- follow-ups sent
- replies
- positive replies
- calls booked
- paid setups closed
- homepage visits
- use-case page visits
- docs or GEO page visits
- social posts published

## Decision Rules

- if reply rate is low, change ICP or subject line before changing everything
- if replies exist but no calls book, tighten the CTA and add sample-lead proof
- if calls book but do not close, improve the offer and first deliverable
- if a social post gets traction, convert it into a permanent public page
- if a public page gets visits but no action, strengthen CTA placement

## Weekly Output

Every week should produce:

- 1 outbound batch
- 2 follow-up batches
- 1 public GEO page or meaningful update
- 2 to 4 social posts
- 1 short weekly memo

## Monthly Review

At month end, review:

- which ICP produced the best reply quality
- which offer produced the best booked-call rate
- which page brought the most useful traffic
- which social format created the most downstream conversations
- what should be standardized in scripts, pages, or workflows
